# 聚宽选股策略使用说明

## 📋 策略概述

这是一个**基本面 + 技术面**结合的量化选股策略，适用于聚宽 (JoinQuant) 平台。

### 策略逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                      策略流程图                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  全部 A 股 → 过滤 ST/新股 → 基本面筛选 → 技术面筛选 → 候选池  │
│                              ↓                              │
│                    收入同比增长 > 0                          │
│                    毛利率同比增长 > 0                         │
│                    费率同比下降                              │
│                              ↓                              │
│                    近 20 日涨幅排序                           │
│                    选取前 10 只                               │
│                              ↓                              │
│                    MACD 金叉 → 买入                          │
│                    MACD 死叉 → 卖出                          │
│                    止盈 30% / 止损 15%                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 🎯 选股条件

### 基本面筛选

| 指标 | 条件 | 说明 |
|------|------|------|
| 营业收入同比增长 | > 0% | 公司业务在扩张 |
| 毛利率同比增长 | > 0% | 盈利能力在提升 |
| 三费费率同比 | 下降 | 费用控制能力增强 |
| 市值 | > 50 亿 | 过滤小市值股票 |

### 技术面筛选

| 指标 | 条件 | 说明 |
|------|------|------|
| 近期动量 | 前 20 日涨幅排名 | 选择强势股 |
| 候选池数量 | 10 只 | 保持适度分散 |

### 交易信号

| 信号类型 | 条件 | 操作 |
|----------|------|------|
| MACD 金叉 | DIF 上穿 DEA | 买入 |
| MACD 死叉 | DIF 下穿 DEA | 卖出 |
| 止损 | 亏损≥15% | 强制卖出 |
| 止盈 | 盈利≥30% | 强制卖出 |

## ⚙️ 参数配置

```python
# 股票池配置
STOCK_POOL_SIZE = 10          # 候选池股票数量
MIN_MARKET_CAP = 50           # 最小市值 (亿)

# 基本面筛选条件
REVENUE_GROWTH_MIN = 0        # 营收同比增长最小值
GROSS_MARGIN_GROWTH_MIN = 0   # 毛利率同比增长最小值
FEE_RATIO_DECLINE_MIN = 0     # 费率同比下降最小值

# 技术面参数
LOOKBACK_DAYS = 20            # 近期涨势观察天数
MACD_FAST = 12                # MACD 快线
MACD_SLOW = 26                # MACD 慢线
MACD_SIGNAL = 9               # MACD 信号线

# 交易参数
POSITION_PER_STOCK = 0.2      # 单只股票最大仓位 (20%)
MAX_POSITIONS = 5             # 最大持仓数量
STOP_LOSS = 0.15              # 止损比例 15%
TAKE_PROFIT = 0.30            # 止盈比例 30%
```

## 🚀 使用步骤

### 1. 登录聚宽平台

访问 https://www.joinquant.com 并登录账号

### 2. 创建新策略

1. 点击「我的策略」→「创建策略」
2. 选择「Python3」语言
3. 选择「股票策略」类型

### 3. 设置回测参数

| 参数 | 推荐值 |
|------|--------|
| 回测时间 | 2025-09-15 至 2026-03-15（半年） |
| 初始资金 | 1,000,000 元 |
| 交易频率 | 日线 |
| 基准指数 | 沪深 300 (000300.XSHG) |
| 滑点比例 | 0.2% |
| 手续费率 | 买入 0.03%，卖出 0.13% |

### 4. 复制代码

将 `joinquant_stock_strategy.py` 中的代码完整复制到策略编辑器

### 5. 运行回测

点击「运行回测」按钮，等待回测完成

### 6. 查看结果

回测完成后查看：
- 收益率曲线
- 夏普比率
- 最大回撤
- 胜率
- 交易记录

## 📊 策略优化建议

### 1. 基本面参数优化

```python
# 更严格的筛选条件（可能减少候选股票数量）
REVENUE_GROWTH_MIN = 0.10      # 营收增长 > 10%
GROSS_MARGIN_GROWTH_MIN = 0.05 # 毛利率增长 > 5%
FEE_RATIO_DECLINE_MIN = 0.02   # 费率下降 > 2%
```

### 2. MACD 参数优化

```python
# 更敏感的 MACD 参数（适合短线）
MACD_FAST = 6
MACD_SLOW = 13
MACD_SIGNAL = 5

# 更稳定的 MACD 参数（适合长线）
MACD_FAST = 24
MACD_SLOW = 52
MACD_SIGNAL = 18
```

### 3. 仓位管理优化

```python
# 根据市场状态调整仓位
def adjust_position_by_market(context):
    # 获取大盘趋势
    market_trend = get_market_trend()
    
    if market_trend == 'bull':
        return 0.15  # 牛市提高仓位
    elif market_trend == 'bear':
        return 0.05  # 熊市降低仓位
    else:
        return 0.10  # 正常仓位
```

### 4. 增加其他技术指标

```python
# RSI 超买超卖
def check_rsi(stock):
    rsi = get_rsi(stock, 14)
    return rsi < 30  # 超卖区域

# KDJ 金叉
def check_kdj(stock):
    k, d, j = get_kdj(stock)
    return k > d and j > 50
```

## ⚠️ 风险提示

1. **历史业绩不代表未来**：回测结果仅供参考，不代表未来收益
2. **市场风险**：极端市场情况下策略可能失效
3. **流动性风险**：小市值股票可能存在流动性问题
4. **过拟合风险**：参数过度优化可能导致实盘效果不佳
5. **交易成本**：实际交易成本可能高于回测假设

## 📝 实盘注意事项

1. **模拟盘测试**：建议先在模拟盘运行 1-3 个月
2. **小资金试水**：实盘初期使用小资金验证
3. **定期复盘**：每周/每月检查策略表现
4. **参数调整**：根据市场变化适度调整参数
5. **风险控制**：设置总资金止损线（如总亏损 20% 停止）

## 🔄 策略更新频率

### 建议更新周期

| 更新类型 | 频率 | 说明 |
|----------|------|------|
| **股票池更新** | 每月 1 次 | 代码已设置为每月 1 号自动更新候选池 |
| **参数回测验证** | 每季度 1 次 | 重新回测验证参数是否仍然有效 |
| **策略逻辑评估** | 每半年 1 次 | 检查策略逻辑是否适应当前市场 |
| **全面优化调整** | 每年 1 次 | 根据年度表现进行系统性优化 |

### 为什么这样设置？

```
┌─────────────────────────────────────────────────────────┐
│              策略更新频率决策框架                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  高频更新 (每周) ❌                                     │
│  → 容易过拟合，追逐短期噪音                              │
│  → 交易成本增加，收益被侵蚀                              │
│                                                         │
│  中频更新 (每月) ✅ 股票池                              │
│  → 财报季更新基本面数据                                  │
│  → 捕捉中期趋势变化                                      │
│  → 平衡稳定性和适应性                                    │
│                                                         │
│  低频更新 (每季/年) ✅ 参数/逻辑                         │
│  → 避免过度优化                                          │
│  → 让策略有足够时间验证                                  │
│  → 减少人为干预偏差                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 需要提前更新的情况

1. **市场风格突变**：如从价值股转向成长股
2. **策略连续失效**：连续 2-3 个月跑输基准
3. **重大政策变化**：如交易规则、财报披露规则变更
4. **黑天鹅事件**：如疫情、金融危机等

### 更新检查清单

```markdown
- [ ] 检查近 1 个月收益率 vs 基准
- [ ] 检查最大回撤是否超过阈值
- [ ] 检查股票池是否有异常集中
- [ ] 检查交易频率是否异常
- [ ] 重新回测最近 3 个月数据
- [ ] 对比同类策略表现
```

## 🔧 常见问题

### Q1: 回测时股票池为空怎么办？
A: 放宽基本面筛选条件，或检查数据获取是否正常

### Q2: 交易信号过于频繁怎么办？
A: 增加 MACD 参数周期，或加入其他过滤条件

### Q3: 收益率不理想怎么办？
A: 
- 检查选股逻辑是否合理
- 优化止盈止损参数
- 考虑加入大盘择时逻辑
- 尝试不同的技术指标组合

### Q4: 如何导出交易记录？
A: 在聚宽平台回测结果页面点击「交易记录」→「导出 Excel」
